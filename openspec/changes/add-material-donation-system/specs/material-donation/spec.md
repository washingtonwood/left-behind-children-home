# 物资捐赠管理规范

## 新增需求

### 需求：物资分类管理
系统必须支持多种物资类型的分类管理。

#### 场景：物资分类枚举
- **当** 系统初始化或管理员配置时
- **那么** 系统必须支持以下物资分类：
  - 衣物（春夏秋冬装、鞋帽、配饰）
  - 书籍（课本、故事书、科普书、工具书）
  - 学习用品（书包、文具、本册、辅导材料）
  - 生活用品（被褥、洗漱用品、餐具）
  - 体育用品（球类、运动器材）
  - 其他物资

### 需求：物资捐赠登记
系统必须允许捐赠者登记物资捐赠信息。

#### 场景：创建物资捐赠
- **当** 捐赠者提交物资捐赠表单
- **并且** 填写物资名称、数量、新旧程度、价值估算
- **并且** 上传物资照片（可选）
- **并且** 选择捐赠方式（邮寄/上门取件）
- **那么** 系统必须创建捐赠记录并生成唯一捐赠编号
- **并且** 返回捐赠编号和物流地址信息

#### 场景：批量物资捐赠
- **当** 捐赠者一次捐赠多种物资
- **那么** 系统必须支持在一个捐赠记录下添加多个物资明细
- **并且** 每个物资明细独立记录类型、数量、价值

#### 场景：捐赠信息验证
- **当** 捐赠者提交的物资信息不完整
- **那么** 系统必须提示缺失字段并阻止提交

### 需求：物资需求匹配
系统必须根据受助儿童的需求展示急需物资清单。

#### 场景：查看物资需求清单
- **当** 捐赠者访问物资捐赠页面
- **那么** 系统必须展示当前急需物资清单
- **并且** 按紧急程度排序（紧急/一般/储备）
- **并且** 显示物资类型、需求数量、已接收数量

#### 场景：按需捐赠
- **当** 捐赠者根据需求清单选择捐赠物资
- **那么** 系统必须在捐赠登记时关联对应需求
- **并且** 自动扣减需求数量

### 需求：物资接收与检验
系统必须记录物资接收和检验结果。

#### 场景：物资接收确认
- **当** 志愿者收到捐赠物资
- **那么** 系统必须更新捐赠状态为"已接收"
- **并且** 记录接收时间和接收人

#### 场景：物资质量检验
- **当** 志愿者检验物资质量
- **那么** 系统必须记录检验结果（合格/不合格/需处理）
- **并且** 不合格物资必须标记原因并通知捐赠者

#### 场景：物资入库
- **当** 物资检验合格
- **那么** 系统必须更新状态为"已入库"
- **并且** 记录入库时间和存储位置

### 需求：捐赠方式管理
系统必须支持多种捐赠方式。

#### 场景：邮寄捐赠
- **当** 捐赠者选择邮寄方式
- **那么** 系统必须提供指定的邮寄地址和联系方式
- **并且** 生成捐赠编号标签供打印

#### 场景：上门取件
- **当** 捐赠者选择上门取件方式
- **并且** 所在城市在服务范围内
- **那么** 系统必须允许预约取件时间
- **并且** 生成取件单

#### 场景：取件服务限制
- **当** 捐赠者所在城市不在服务范围
- **并且** 选择上门取件
- **那么** 系统必须提示不支持并引导选择邮寄方式

### 需求：捐赠查询
捐赠者必须能够查询自己的捐赠记录。

#### 场景：查询捐赠记录
- **当** 捐赠者提供捐赠编号或登录账户
- **那么** 系统必须显示捐赠详情
- **并且** 包含物资清单、当前状态、物流信息

#### 场景：捐赠状态通知
- **当** 捐赠状态发生变化
- **那么** 系统必须通过站内消息或邮件通知捐赠者（如果提供了联系方式）
