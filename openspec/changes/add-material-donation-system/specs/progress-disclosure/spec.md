# 帮扶进度信息公开规范

## 新增需求

### 需求：进度记录创建
志愿者和管理员必须能够创建和发布帮扶进度记录。

#### 场景：创建进度记录
- **当** 志愿者完成一次帮扶活动
- **那么** 系统必须允许创建进度记录
- **并且** 必须填写：
  - 关联的儿童 ID
  - 关联的项目 ID（如有）
  - 活动日期
  - 活动类型（家访、学业辅导、心理关怀、物资发放等）
  - 活动描述（文字，最多 2000 字）
  - 参与志愿者
- **并且** 可选上传：
  - 活动照片（最多 20 张）
  - 活动视频（最多 3 个，每个 < 100MB）
  - 相关文档

#### 场景：进度草稿保存
- **当** 志愿者填写进度记录但未完成
- **那么** 系统必须支持保存为草稿
- **并且** 允许后续继续编辑
- **并且** 草稿对公众不可见

#### 场景：进度发布
- **当** 志愿者完成进度记录并提交发布
- **那么** 系统必须先进行内容审核
- **并且** 审核通过后对公众可见
- **并且** 审核不通过时返回修改意见

### 需求：进度内容展示
系统必须以多种形式展示帮扶进度，确保公众直观了解。

#### 场景：时间轴展示
- **当** 用户查看某个儿童的帮扶进度
- **那么** 系统必须以垂直时间轴展示所有帮扶活动
- **并且** 按时间倒序排列（最新在前）
- **并且** 每个节点显示：
  - 活动日期
  - 活动类型（图标 + 文字）
  - 活动简介（最多 100 字）
  - 参与志愿者
  - 照片缩略图（如有）

#### 场景：卡片式展示
- **当** 用户查看项目帮扶进度
- **那么** 系统必须以卡片网格展示
- **并且** 每张卡片显示：
  - 活动封面图
  - 活动标题
  - 活动日期
  - 简短描述
- **并且** 点击卡片查看详情

#### 场景：地图可视化
- **当** 用户查看帮扶覆盖范围
- **那么** 系统必须在地图上标注帮扶地点
- **并且** 标注点显示：
  - 地点名称
  - 帮扶次数
  - 最近活动时间
- **并且** 点击标注点查看该地帮扶详情

#### 场景：数据统计卡片
- **当** 用户查看帮扶统计
- **那么** 系统必须显示关键指标卡片：
  - 累计帮扶次数
  - 覆盖儿童人数
  - 参与志愿者人数
  - 发放物资数量
  - 帮扶覆盖地区数

### 需求：进度详情页
系统必须提供详细的帮扶活动信息。

#### 场景：查看进度详情
- **当** 用户点击某个进度记录
- **那么** 系统必须显示完整信息：
  - 活动标题和日期
  - 活动类型
  - 参与志愿者（姓名 + 头像）
  - 活动详细描述
  - 活动照片墙（可放大查看）
  - 活动视频（可播放）
  - 相关附件（可下载）
  - 关联的受助儿童（脱敏信息）
  - 关联的物资捐赠（如有）

#### 场景：照片轮播
- **当** 进度记录包含多张照片
- **那么** 系统必须提供照片轮播组件
- **并且** 支持左右切换、全屏查看
- **并且** 显示照片总数和当前索引

#### 场景：视频播放
- **当** 进度记录包含视频
- **那么** 系统必须提供视频播放器
- **并且** 支持播放、暂停、进度条
- **并且** 移动端适配良好

### 需求：进度分类与筛选
系统必须支持按不同维度查看帮扶进度。

#### 场景：按儿童查看进度
- **当** 用户选择某个受助儿童
- **那么** 系统必须显示该儿童的所有帮扶进度
- **并且** 按时间线展示
- **并且** 统计该儿童累计帮扶次数和类型

#### 场景：按项目查看进度
- **当** 用户选择某个救助项目
- **那么** 系统必须显示该项目的所有帮扶进度
- **并且** 按活动日期分组
- **并且** 统计项目进展（计划人数 vs 已覆盖人数）

#### 场景：按活动类型筛选
- **当** 用户选择活动类型（如"学业辅导"）
- **那么** 系统必须只显示该类型的帮扶进度
- **并且** 显示该类型的统计总数

#### 场景：按时间范围筛选
- **当** 用户选择时间范围（如"最近 30 天"）
- **那么** 系统必须只显示该时间范围内的进度
- **并且** 提供快捷选项（本周/本月/本年/自定义）

### 霜求：帮扶效果评估
系统必须定期发布帮扶效果报告。

#### 场景：个人效果报告
- **当** 志愿者或管理员为某个儿童创建效果报告
- **那么** 系统必须生成包含以下内容的报告：
  - 儿童基本信息（脱敏）
  - 帮扶时长（月数）
  - 帮扶活动统计（各类型次数）
  - 学业改善情况（成绩对比、学习态度变化）
  - 生活改善情况（居住条件、营养状况）
  - 心理变化情况（开朗程度、自信心）
  - 志愿者评语
  - 照片对比（帮扶前后）
  - 下一步帮扶计划

#### 场景：项目效果报告
- **当** 项目结束或阶段性总结
- **那么** 系统必须生成项目效果报告
- **并且** 包含：
  - 项目总体目标 vs 实际完成
  - 覆盖儿童人数统计
  - 物资发放统计
  - 资金使用统计
  - 志愿者参与统计
  - 典型案例展示（3-5 个）
  - 存在问题和改进建议

#### 场景：报告发布通知
- **当** 效果报告发布
- **那么** 系统必须通知：
  - 相关捐赠者（捐赠过该儿童或项目）
  - 关注该地区的志愿者
  - 订阅了更新的用户
- **并且** 通知包含报告链接和关键数据摘要

### 霜求：进度更新提醒
系统必须主动推送进度更新给关注用户。

#### 场景：新进度发布通知
- **当** 有新的帮扶进度发布
- **那么** 系统必须推送通知给：
  - 捐赠过该儿童的用户
  - 关注该项目的用户
  - 参与过该地区帮扶的志愿者
- **并且** 通知内容包含：
  - 进度标题
  - 活动日期
  - 简短描述（50 字以内）
  - 封面图
  - 查看详情链接

#### 场景：订阅提醒
- **当** 用户订阅某个儿童或项目的进度更新
- **然后** 系统必须在每次有新进度时发送通知
- **并且** 用户可以随时取消订阅

### 霜求：隐私保护与敏感信息处理
系统必须在公开进度时保护受助儿童隐私。

#### 场景：儿童信息脱敏
- **当** 发布帮扶进度时
- **那么** 系统必须对儿童信息进行脱敏处理：
  - 姓名显示为"小明"（化名）
  - 或显示为"小*同学"
  - 地址精确到乡镇级，不显示门牌号
  - 联系方式完全不显示

#### 场景：照片隐私保护
- **当** 上传帮扶活动照片
- **那么** 系统必须要求对儿童面部进行隐私处理：
  - 使用马赛克遮挡面部
  - 或使用表情贴纸遮挡
  - 或使用不露正面的拍摄角度
- **并且** 管理员审核时必须检查隐私处理是否到位

#### 场景：敏感信息审核
- **当** 志愿者提交进度记录
- **那么** 系统必须进行敏感信息审核
- **并且** 检测并过滤：
  - 详细家庭地址
  - 电话号码
  - 身份证号
  - 银行账户
  - 其他个人隐私信息
- **并且** 发现敏感信息时标记并要求修改

### 霜求：进度互动与反馈
系统必须允许用户对帮扶进度进行互动。

#### 场景：点赞和鼓励
- **当** 用户查看帮扶进度
- **那么** 系统必须允许点击"点赞"或"送爱心"
- **并且** 显示点赞数量
- **并且** 防止重复点赞（同一用户只能点赞一次）

#### 场景：评论留言
- **当** 用户查看帮扶进度
- **那么** 系统必须允许发表评论留言
- **并且** 评论必须先经过审核
- **并且** 禁止评论功能（如关闭评论）

#### 场景：分享功能
- **当** 用户查看帮扶进度
- **那么** 系统必须提供分享功能
- **并且** 支持分享到：
  - 微信（生成二维码）
  - 微博
  - QQ
  - 复制链接
- **并且** 分享时生成预览图和摘要

### 霜求：数据统计与可视化
系统必须提供帮扶数据的统计分析。

#### 场景：帮扶趋势图表
- **当** 用户查看帮扶统计
- **那么** 系统必须展示趋势图表：
  - 帮扶次数随时间变化（折线图）
  - 各类型帮扶占比（饼图）
  - 各地区帮扶分布（地图热力图）
  - 物资发放统计（柱状图）

#### 场景：志愿者贡献排行
- **当** 查看志愿者统计
- **那么** 系统必须显示志愿者贡献排行
- **并且** 指标包括：
  - 参与帮扶次数
  - 服务时长
  - 发布进度记录数
- **并且** 支持 Top 10 展示

#### 场景：捐赠者影响力排行
- **当** 查看捐赠者统计
- **那么** 系统必须显示捐赠者影响力排行
- **并且** 指标包括：
  - 捐赠次数
  - 捐赠物资数量
  - 帮扶儿童人数
- **并且** 匿名显示（如"爱心捐赠者 A"）

### 霜求：移动端适配
帮扶进度展示必须在移动设备上有良好体验。

#### 场景：移动端时间轴展示
- **当** 用户在手机上查看进度时间轴
- **那么** 系统必须优化布局：
  - 采用紧凑型时间轴设计
  - 照片使用网格布局
  - 支持左右滑动查看更多

#### 场景：移动端图片加载优化
- **当** 移动端用户查看包含大量照片的进度
- **那么** 系统必须：
  - 使用懒加载（滚动到才加载）
  - 使用缩略图，点击加载大图
  - 压缩图片大小（< 500KB）
