# 帮扶对象推荐系统规范

## 新增需求

### 需求：智能推荐引擎
系统必须根据多维度指标为捐赠者推荐最需要帮助的儿童。

#### 场景：生成推荐列表
- **当** 捐赠者访问帮扶对象推荐页面
- **那么** 系统必须根据推荐算法生成推荐对象列表
- **并且** 按综合得分从高到低排序
- **并且** 每页显示 12-20 个推荐对象
- **并且** 每个推荐显示：姓名（脱敏）、年龄、地区、需求描述、推荐理由

#### 场景：推荐理由展示
- **当** 系统展示推荐对象时
- **那么** 每个推荐必须说明推荐理由
- **并且** 推荐理由包含：
  - 紧急程度说明（如：家庭困难，急需学习用品）
  - 需求匹配度（如：您捐赠的书籍正是孩子急需的）
  - 帮扶效果预期（如：每月 200 元可改善学习条件）

### 需求：推荐权重计算
系统必须使用可配置的权重算法计算推荐得分。

#### 场景：计算推荐得分
- **当** 系统计算儿童的综合推荐得分
- **那么** 必须综合考虑以下因素：
  - **紧急程度**（权重 30%）：根据家庭困难程度、需求紧急性
  - **需求匹配度**（权重 25%）：根据捐赠者偏好与儿童需求的匹配程度
  - **时间紧迫性**（权重 20%）：等待帮扶的时间长度
  - **地区优先级**（权重 15%）：偏远地区、贫困地区优先
  - **帮扶效果**（权重 10%）：历史帮扶效果数据

#### 场景：紧急程度评估
- **当** 评估儿童紧急程度时
- **那么** 系统必须考虑：
  - 家庭年收入水平
  - 家庭成员健康状况
  - 居住条件
  - 监护人情况（留守儿童、单亲家庭等）

#### 场景：需求匹配度计算
- **当** 捐赠者选择物资类型或帮扶方向
- **那么** 系统必须优先推荐有相应需求的儿童
- **并且** 匹配度越高，推荐排名越靠前

### 需求：多维度筛选
系统必须允许捐赠者按条件筛选帮扶对象。

#### 场景：按地区筛选
- **当** 捐赠者选择一个或多个地区
- **那么** 系统必须筛选出该地区的儿童
- **并且** 支持省份、城市、县级筛选

#### 场景：按年龄段筛选
- **当** 捐赠者选择年龄段（如 6-12 岁）
- **那么** 系统必须筛选出符合年龄段的儿童

#### 场景：按需求类型筛选
- **当** 捐赠者选择需求类型
- **那么** 系统必须筛选出有对应需求的儿童
- **并且** 支持多选（如同时有教育需求和生活需求）

#### 场景：按帮扶进度筛选
- **当** 捐赠者选择帮扶进度状态
- **那么** 系统必须筛选出对应状态的儿童
- **并且** 支持筛选：未开始、进行中、已完成

### 需求：个性化推荐
系统必须根据捐赠者的历史行为和偏好提供个性化推荐。

#### 场景：基于历史捐赠推荐
- **当** 捐赠者有历史捐赠记录
- **那么** 系统必须优先推荐与其历史捐赠类型匹配的儿童
- **并且** 推荐理由中提及历史捐赠行为

#### 场景：基于关注地区推荐
- **当** 捐赠者关注特定地区
- **那么** 系统必须优先推荐该地区的儿童

#### 场景：新用户默认推荐
- **当** 新用户首次访问推荐页面
- **那么** 系统必须按紧急程度推荐最需要帮助的儿童
- **并且** 提示可以设置偏好以获得更精准的推荐

### 需求：推荐管理
管理员必须能够配置推荐规则和查看推荐效果。

#### 场景：调整推荐权重
- **当** 管理员修改推荐算法的权重配置
- **那么** 系统必须立即应用新权重到后续推荐
- **并且** 保留权重变更历史

#### 场景：查看推荐日志
- **当** 管理员查看推荐日志
- **那么** 系统必须显示：
  - 推荐时间
  - 推荐对象列表
  - 捐赠者信息（匿名化）
  - 推荐理由
  - 后续转化情况（是否捐赠）

#### 场景：推荐效果统计
- **当** 管理员查看推荐效果统计
- **那么** 系统必须显示：
  - 推荐转化率（推荐后实际捐赠的比例）
  - 各地区推荐效果
  - 各需求类型推荐效果
  - 最受欢迎的推荐对象

### 需求：推荐去重与优化
系统必须避免重复推荐同一对象，并优化推荐多样性。

#### 场景：已捐赠对象不再推荐
- **当** 捐赠者已对某个儿童进行捐赠
- **那么** 系统必须在后续推荐中排除该儿童
- **除非** 捐赠者明确查看全部儿童

#### 场景：推荐多样性
- **当** 系统生成推荐列表
- **那么** 必须避免推荐过于集中在同一地区或同一类型
- **并且** 确保列表包含多样化的帮扶对象

#### 场景：冷启动处理
- **当** 系统中暂无符合筛选条件的儿童
- **那么** 系统必须提示"暂无符合条件的儿童"
- **并且** 提供放宽筛选条件的建议
