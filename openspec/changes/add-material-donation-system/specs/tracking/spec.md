# 物资流向追踪规范

## 新增需求

### 需求：全流程追踪
系统必须记录物资从捐赠到发放的每一个关键节点。

#### 场景：追踪流程定义
- **当** 系统记录物资流向时
- **然后** 必须包含以下关键节点：
  1. **已创建** - 捐赠信息登记
  2. **已支付** - 捐赠者完成支付（如适用）
  3. **待发货** - 捐赠者已发货，等待接收
  4. **已接收** - 志愿者收到物资
  5. **检验中** - 物资质量检验
  6. **已入库** - 检验合格，入库存储
  7. **分配中** - 匹配受助儿童
  8. **已发货** - 发往受助儿童
  9. **已签收** - 受助儿童签收
  10. **已完成** - 反馈照片已上传

#### 场景：节点时间戳记录
- **当** 物资状态发生变化
- **那么** 系统必须记录精确的时间戳
- **并且** 记录操作人员信息
- **并且** 记录节点备注说明

### 需求：追踪节点存证
系统必须在关键节点要求上传照片作为凭证。

#### 场景：接收节点拍照
- **当** 志愿者接收物资
- **那么** 系统必须要求上传物资包裹照片
- **并且** 照片必须清晰显示包裹外观和捐赠编号

#### 场景：检验节点拍照
- **当** 志愿者完成物资检验
- **那么** 系统必须上传物资开箱检验照片
- **并且** 展示物资种类和数量

#### 场景：签收节点拍照
- **当** 受助儿童签收物资
- **那么** 系统必须上传儿童签收照片
- **并且** 照片中儿童面部必须做隐私保护处理（马赛克/遮挡）
- **或者** 使用不露正面的角度拍摄

#### 场景：签收视频反馈（可选）
- **当** 受助儿童或监护人愿意提供视频反馈
- **那么** 系统必须支持视频上传（大小限制 50MB）
- **并且** 视频格式限制为 MP4、MOV

### 霜求：物流信息集成
系统必须支持集成快递公司物流信息。

#### 场景：快递单号录入
- **当** 捐赠者选择邮寄捐赠
- **并且** 物资已寄出
- **那么** 系统必须允许捐赠者或志愿者录入快递单号
- **并且** 选择快递公司

#### 场景：物流信息自动查询
- **当** 系统记录了快递单号和快递公司
- **那么** 系统必须定期查询物流状态（每小时一次）
- **并且** 自动更新追踪节点
- **并且** 物流异常时提醒管理员

#### 场景：物流信息展示
- **当** 捐赠者查看物资追踪
- **并且** 存在物流信息
- **那么** 系统必须显示物流时间线
- **并且** 包含：时间、地点、状态描述

### 需求：追踪时间线展示
系统必须以直观的时间线方式展示物资流转过程。

#### 场景：查看追踪时间线
- **当** 捐赠者查询物资追踪
- **那么** 系统必须展示垂直时间线
- **并且** 每个节点显示：
  - 节点名称（如"已接收"）
  - 时间（精确到分钟）
  - 操作人员
  - 节点照片/视频（如有）
  - 备注说明
- **并且** 已完成节点显示为绿色
- **并且** 当前节点显示为蓝色并高亮
- **并且** 未完成节点显示为灰色

#### 场景：追踪进度百分比
- **当** 展示追踪时间线时
- **那么** 系统必须显示整体进度百分比
- **并且** 根据已完成的节点数计算（如 6/10 节点 = 60%）

### 需求：异常处理
系统必须处理物资流转过程中的异常情况。

#### 场景：物资丢失处理
- **当** 物流信息显示异常或物资丢失
- **那么** 系统必须标记为"异常"状态
- **并且** 通知管理员和捐赠者
- **并且** 记录异常原因和处理方案

#### 场景：物资不合格处理
- **当** 检验发现物资不合格
- **那么** 系统必须更新状态为"检验不合格"
- **并且** 记录不合格原因
- **并且** 通知捐赠者协商处理方式

#### 场景：签收失败处理
- **当** 发往受助儿童但签收失败（如地址错误、儿童搬迁）
- **那么** 系统必须标记为"签收失败"
- **并且** 通知管理员重新确认信息
- **并且** 更新正确信息后重新发货

### 需求：追踪权限管理
系统必须控制追踪信息的访问权限。

#### 场景：捐赠者查看权限
- **当** 捐赠者查询自己捐赠的追踪信息
- **那么** 系统必须允许查看完整追踪时间线
- **并且** 允许查看所有节点照片/视频

#### 场景：公众查看权限
- **当** 未登录公众查看追踪信息
- **那么** 系统必须允许查看公开的追踪信息
- **并且** 隐藏敏感信息（如详细地址、联系方式）
- **并且** 受助儿童照片做隐私处理

#### 场景：管理员权限
- **当** 管理员查看追踪信息
- **那么** 系统必须允许查看所有详细信息
- **并且** 允许编辑追踪节点
- **并且** 允许添加备注和说明

### 需求：追踪提醒
系统必须在关键节点主动通知相关人员。

#### 场景：物资接收提醒
- **当** 志愿者接收物资并录入系统
- **那么** 系统必须发送通知给捐赠者
- **并且** 通知内容包含：接收时间、物资状态、下一步安排

#### 场景：签收提醒
- **当** 受助儿童签收物资
- **那么** 系统必须发送通知给捐赠者
- **并且** 附上签收照片预览

#### 场景：异常提醒
- **当** 物资流转出现异常
- **那么** 系统必须立即通知管理员
- **并且** 说明异常类型和建议处理方式

### 需求：追踪统计
系统必须提供追踪数据的统计分析。

#### 场景：平均流转时间统计
- **当** 管理员查看追踪统计
- **那么** 系统必须显示物资平均流转时间
- **并且** 按阶段统计（如：平均 3 天到达，5 天完成检验）

#### 场景：异常率统计
- **当** 管理员查看追踪统计
- **那么** 系统必须显示异常率
- **并且** 按异常类型分类统计

#### 场景：地区流转效率对比
- **当** 管理员查看不同地区的流转效率
- **那么** 系统必须按地区统计平均完成时间
- **并且** 标识效率较低的地区
